# Лабораторная работа № 1. Классы

## Общее задание

- Реализовать классы согласно варианту.
- Необходимое по варианту поведение допускается реализовать (выбор на ваше усмотрение):
  - с помощью функций;
  - с помощью методов.
- Покрыть разработанный функционал тестами с использованием [`googletest`](https://google.github.io/googletest/).
- Динамическую память использовать запрещено.
- Работа должна выполняться в [`git`](https://git-scm.com/)-репозитории.

### Реализуемые классы

Реализовать два класса с условными названиями `Item` и `Container`.
Реальные названия и поведение классов определяются вариантом.
Рекомендуемое название для `Item`-а приведено в первом предложении описания варианта.
Название для `Container`-а подобрать самостоятельно по ситуации.
Помимо `Item`-а и `Container`-а в некоторых вариантах можно и нужно реализовать вспомогательные классы по мере необходимости.

Например, для варианта [Chess1](#вариант-chess):
- `Item` - это класс `ChessPiece` (шахматная фигура);
- `Container` - это класс `ChessBoard` (шахматная доска);
- вспомогательный класс - это класс `Point` для описания координат фигуры на шахматной доске.

### Item-класс

Представляет из себя класс, описывающий различные виды объектов.
Минимальный набор видов, поддержка которых должна быть реализована, приведён в описании варианта.
По своему усмотрению можно добавить новые виды.

*Формально каждый вид можно описать отдельным классом, например, `Item1`, `Item2` и т. д.*
Однако по условию необходимо хранить набор (массив) объектов различного вида для возможности выполнения каких-либо единообразных операций.
Напомним, что в массиве можно хранить только объекты одинакового типа.
Механизм наследования позволяет красиво решить данную проблему, но будет изучен и применён только в одной из последующих лабораторных работ.
Поэтому в данной работе необходимо поступить следующим образом.

*Фактически все виды описать одним классом `Item` с полем `enum`-типа, которое и определяет вид объекта.*
Также в классе объявить полный набор полей, необходимых для описания каждого используемого вида.
В конкретном экземпляре класса потенциально может использоваться только некоторое подмножество полей - в зависимости от значения поля `enum`-типа, которое определяет конкретный вид объекта.

### Container-класс

- Хранит набор объектов `Item` в виде статического массива.
- Реализовать методы для работы с массивом:
  - получение количества элементов;
  - получение или установка элемента по указанному индексу через оператор `[]`;
  - вставка элемента по указанному индексу;
  - удаление элемента по указанному индексу.
- В случае выхода индекса за допустимые пределы бросать исключение.

## Варианты

### Демонстрационный вариант

Математические функции от одного аргумента `Function`.

- Поддерживаемые виды
  - Косинусоидальная функция $f(x) = a * \cos(x)$
  - Синусоидальная функция $f(x) = a * \sin(x)$
- Состояние
  - Вид математической функции и в зависимости от вида - её параметры
- Поведение
  - Вычисление значения функции от заданного аргумента $x$
  - Получение производной функции
  - Поиск в наборе функции с максимальным значением при заданном аргументе

  - Поиск в наборе фигуры с минимальной площадью
