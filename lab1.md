### Реализуемые классы

Реализовать два класса с условными названиями `Item` и `Container`.
Реальные названия и поведение классов определяются вариантом.
Рекомендуемое название для `Item` приведено в первом предложении описания варианта.
Название для `Container` подобрать самостоятельно по ситуации.
Помимо `Item` и `Container` в некоторых вариантах можно и нужно реализовать вспомогательные классы по мере необходимости.

Например, для варианта [Chess1](#вариант-chess1):
- `Item` - это класс `ChessPiece` (шахматная фигура);
- `Container` - это класс `ChessBoard` (шахматная доска);
- вспомогательный класс - это класс `Point` для описания координат фигуры на шахматной доске.

### Класс `Item`

Представляет из себя класс, описывающий различные виды объектов.
Минимальный набор видов, поддержка которых должна быть реализована, приведён в описании варианта.
По своему усмотрению можно добавить новые виды.

*Формально каждый вид можно описать отдельным классом, например, `Item1`, `Item2` и т. д.*
Однако по условию необходимо хранить набор (массив) объектов различного вида для возможности выполнения каких-либо единообразных операций.
Напомним, что в массиве можно хранить только объекты одинакового типа.
Механизм наследования позволяет красиво решить данную проблему, но будет изучен и применён только в одной из последующих лабораторных работ.
Поэтому в данной работе необходимо поступить следующим образом.

*Фактически все виды описать одним классом `Item` с полем `enum`-типа, которое и определяет вид объекта.*
Состояние объекта класса `Item` описывается видом объекта (`enum`-поле) и набором полей, некоторые из которых потенциально могут использоваться только объектами определённого вида.
Минимальный набор полей определяется требуемым состоянием, которе приведено в описании варианта.
По своему усмотрению можно добавить дополнительные поля.

### Класс `Container`

- Хранит набор объектов `Item` в виде статического массива
- Реализовать методы для работы с массивом:
  - получение количества элементов;
  - получение или установка элемента по указанному индексу через оператор `[]`;
  - вставка элемента по указанному индексу;
  - удаление элемента по указанному индексу.
- В случае выхода индекса за допустимые пределы бросать исключение
- При вставке элемента в ситуации, когда массив уже полностью заполнен, бросать исключение

### Вариант 5: Citizen

Человек из социального слоя `Citizen`.

- Поддерживаемые виды
  - Школьник
  - Студент
  - Пенсионер
- Состояние
  - ФИО

    - Для студентов:
      - 0, если средний балл меньше 3.5;
      - от 0.5\*МРОТ до 0.8\*МРОТ при среднем балле от 3.5 до 5 (зависимость линейная).
    - Для пенсионера:
      - 0, если стаж меньше 5 лет;
      - от 1\*МРОТ до 4\*МРОТ при стаже от 5 до 30 лет (зависимоть линейная);
      - 5\*МРОТ при стаже более 30 лет.

### Вариант 6: Class

Университетские занятия `Class`.

- Поддерживаемые виды
  - Лекции
  - Лабораторные работы
  - Практики
- Состояние
  - Название дисциплины
  - Количество часов $h$
- Функционал
  - Для отдельного занятия - вычисление количества оплачиваемых преподавателю часов $p$
    - Входные аргументы
      - Количество групп $n$
      - Количество подгрупп $m$ (обычно $m = 2 * n$, но так как для маленьких групп разделение на подгруппы может не производиться, полагать, что $n \leq m \leq 2 \cdot n$)
    - Порядок расчёта
      - Лекции читаются всему потоку сразу, поэтому для лекций $p = h$
      - Практики читаются по группам, поэтому для практик $p = n \cdot h$
      - Лабораторные ведутся по подгруппам, поэтому для лабораторных $p = m \cdot h$
  - Для набора занятий - вычисление полного количества оплачиваемых часов для заданной дисциплины (по заданному названию, $n$ и $m$)
  - Для набора занятий - определение названия первой дисциплины с максимальным количеством оплачиваемых часов по всем занятиям (по заданному $n$ и $m$)

  - Объем двигателя $V$
  - Только для грузовых автомобилей: тоннаж $T$
- Функционал
  - Расчёт транспортного налога по заданной базовой ставке $S$:
    - для легкового автомобиля: $S \cdot V$;
    - для грузового автомобиля: $S \cdot V \cdot (T/2 + 1)$;
    - для мотоцикла: $0.3 \cdot S \cdot V$.
  - Поиск в наборе первого транспорта с наибольшим значением транспортного налога

### Вариант 18: Vehicle2

Транспортные средства `Vehicle`.

- Поддерживаемые виды
  - Железнодорожный транспорт
  - Водный транспорт
  - Воздушный транспорт
- Состояние
  - Модель транспорта
  - Базовый тариф перевозки $k$ \[руб/(кг\*км)\]
  - Только для водного транспорта: поощряющий коэффициент $0.9 \leq a \leq 1$ за большие расстояния доставки
  - Только для воздушного транспорта: тип двигателя (турбовинтовой или реактивный)
- Функционал
  - Расчёт цены транспортировки
    - Входные аргументы
      - Масса груза $m$ \[кг\]
      - Расстояние $d$ \[км\]
    - Алгоритм расчёта:
      - Для железнодорожного транспорта: $k \cdot m \cdot d$
      - Для водного транспорта: $k \cdot m \cdot d \cdot a^{d / 5000}$
      - Для воздушного транспорта: $\omega \cdot k \cdot m \cdot d$
        - Для турбовинтового транспорта:
          - $\omega = 0.5$, если $d < 1000$;
          - $\omega = 1.5$, иначе.
        - Для реактивного транспорта:
          - $\omega = 1.5$, если $d < 1000$;
          - $\omega = 0.5$, иначе.
  - Поиск в наборе первого транспорта с минимальной ценой транспортировки при заданных $m$ и $d$
