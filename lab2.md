# Лабораторная работа № 2. Управление ресурсами

Напомним, что в первой лабораторной работе были реализованы классы `Item` и `Container`.
Вторая лабораторная работа представляет из себя переделку первой.

Все изменения, которые необходимы сделать, перечислены ниже.
Основной пласт изменений здесь приходится на класс `Container` и разработку консольного приложения.

### Класс `Item`

- Добавить метод `clone` для создания глубокой копии объекта в динамической памяти
- Добавить метод `print` для распечатки подробного состояния объекта в указанный поток вывода (данный метод понадобится для консольного приложения)

### Класс `Container`

- Статический массив должен быть заменён на динамический массив
  - Ограничение на максимальное количество объектов (имеющее смысл только для статических массивов) должно быть удалено
- Контейнер теперь должен работать не с объектами, а с указателями на объекты (массив объектов заменяется на массив указателей)
- Контейнер должен корректно управлять всей памятью
  - Операции копирования должны создавать корректную глубокую копию
  - Деструктор должен освобождать память, выделенную как под массив, так и под сами объекты
  - В процессе работы не должно быть утечек памяти или висячих указателей

### Разработка консольного приложения

Сейчас решение состоит из двух проектов: динамической библиотеки и приложения с тестами.
Добавьте третий проект в виде консольного приложения, которое использует разработанную динамическую библиотеку.
Консольное приложение должно представлять из себя меню со следующими (как минимум) возможными действиями:
- вставить элемент в список по указанному индексу;
- удалить элемент из списка по указанному индексу;
- вывести список на экран;
- выполнить поиск в списке объекта по определённому критерию (см. соответствующий пункт секции "Функционал" своего варианта в задании на первую лабораторную работу);

Контроль ввода допускается не делать, но его реализация приветствуется.

### Рекомендации по выполнению

Ни в коем случае не выполняйте рефакторинг классов (`Item`, `Container`) и разработку нового приложения одновременно:
- сначала переделайте классы `Item` и `Container`, обновите тесты и убедитесь что всё работает;
- работоспособный код после рефакторинга зафиксируйте коммитом;
- только после этого приступайте к разработке консольного приложения.
