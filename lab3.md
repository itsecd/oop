# Лабораторная работа № 3. Наследование и полиморфизм

Напомним, что во второй лабораторной работе объекты `Item` переехали в динамическую память, а `Container` взял на себя функции по корректному управлению этой памятью.
Третья лабораторная работа представляет из себя переделку второй.

Все изменения, которые необходимы сделать, перечислены ниже.
Основной пласт изменений здесь приходится на класс `Item`.
Изменения в классе `Container` в основном связаны с его значительным упрощением.

### Класс `Item`

- Декомпозировать класс `Item` в иерархию классов с вынесением общего функционала в базовый класс
  - Поле `enum`-типа, отвечающее за вид объекта, естественным образом отовсюду уйдёт

### Класс `Container`

- Реализовать "правило нуля", делегировав всё управление памятью стандартным классам
  - Управление памятью под массив должно быть делегировано стандартному контейнеру `std::vector<T>`
  - Управление памятью под объекты должно быть делегировано "умным" указателям `std::shared_ptr<T>`
  - Несмотря на "правило *нуля*", конструктор копирования и оператор присваивания копированием останутся, т. к. по условию необходимо создавать глубокие копии
  - Деструктор естественным образом уйдёт

### Рекомендации по выполнению

Работу рекомендуется выполнять в два этапа:
- сначала выполнить декомпозицию класса `Item`, исправить тесты и убедиться, что всё работает;
- и только после успешной декомпозиции с обязательной проверкой её успешности приступить к рефакторингу класса `Container`.
